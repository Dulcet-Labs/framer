**1. The Problem:**
Building Farcaster Frames and Base Mini Apps currently requires a fragmented workflow. A developer has to juggle their code editor, a browser window (localhost), a browser wallet extension, and often a mobile simulator. This constant context switching kills flow and slows down the iteration loop. Web2 devs have live previews; Web3 devs are still alt-tabbing.

**2. The Solution:**
**Base Frame Preview** is a lightweight VS Code extension that embeds a mobile-first simulator directly into the developer's sidebar.
*   **Instant Feedback:** See your Frame/Mini App update in real-time as you type code.
*   **Mobile-First Environment:** Forces developers to build for the mobile screen constraint from Day 1.
*   **Zero Setup:** No external apps. It simply renders your `localhost` inside a controlled environment.

**3. Current Status (MVP):**
We have shipped the V1 Alpha. It successfully renders local Next.js/Vite environments inside a mobile skin within the VS Code sidebar.
