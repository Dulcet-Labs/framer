# Product Name: FramerIDE
### **Tagline:** The Native Workbench for the Onchain Era.

### **The Vision**
FramerIDE is a high-performance, modular desktop development environment designed to replace the fragmented "VS Code + Browser + Wallet + Terminal" workflow. It is built to make building Mini Apps, Farcaster Frames, and Onchain DApps as fast and intuitive as building standard web pages.

While generic editors are built for *code*, FramerIDE is built for *state*. It understands blockchains, wallets, and smart contracts natively.

## üíé Core Module: Base Studio
**Base Studio** is the flagship environment inside FramerIDE, specifically engineered for the Base (Coinbase) ecosystem. It eliminates the friction of Web3 development by providing a zero-config environment for building Farcaster Frames and Base Mini Apps.

### **Key Features of Base Studio:**

#### **1. The "Live-Context" Simulator**
Standard browsers can‚Äôt test Mini Apps properly. Base Studio features a split-pane **Context Simulator** that mimics the exact environment of a user‚Äôs mobile device.
*   **Farcaster Frame Mode:** Previews exactly how your Frame looks in a Warpcast feed. It handles OpenGraph tags, button clicks, and redirects natively without needing `ngrok`.
*   **Coinbase Wallet Mode:** Simulates the Mini App experience inside the Coinbase Wallet/Exchange app.
*   **Instant Reload:** Hot Module Replacement (HMR) that maintains wallet state (you don't get logged out when you save a file).

#### **2. Native Wallet Injection (The "Mock" Bridge)**
Forget installing browser extensions or scanning QR codes to test local changes.
*   **Virtual Provider:** Base Studio injects a mocked `window.ethereum` and `minikit` provider directly into the preview.
*   **One-Click Signatures:** Test complex user flows (Sign, Approve, Swap) by clicking "Accept" in the IDE‚Äôs debug panel. No real wallet popup required.
*   **Transaction Inspector:** See the raw data of every transaction request before it hits the network.

#### **3. Built-in "Base Local" (Powered by Anvil)**
Developers shouldn‚Äôt have to run external terminals just to start a blockchain.
*   **Zero-Config Chain:** When you open Base Studio, it silently spins up a local fork of the Base blockchain (using a bundled Rust/Anvil instance).
*   **Infinite Faucet:** Your dev wallet is pre-funded with 10,000 simulated ETH and USDC.
*   **Contract Hot-Deploy:** Modify a smart contract in the editor and auto-deploy it to your local chain instantly to test frontend integration.

#### **4. One-Command Ship**
*   **Smart Deployment:** When ready, Base Studio handles the upload to IPFS/Arweave and verifies your smart contracts on BaseScan automatically.
*   **Frame Validator:** A built-in linter that checks your Frame against Farcaster specs (image size, button logic) before you push to production.

## üèó Technical Architecture (The Engine)
FramerIDE is built for speed. It discards the bloat of standard Electron apps in favor of a next-gen stack.

*   **Core Shell:** **Tauri (Rust)**.
    *   *Benefit:* Uses the OS native webview (WebKit/WebView2). The installer is <15MB, and RAM usage is minimal compared to VS Code.
*   **Backend Logic:** **Rust**.
    *   *Benefit:* Handles the heavy lifting (local blockchain nodes, file watching, compilation) with near-native metal performance.
*   **Editor Surface:** **Monaco Editor**.
    *   *Benefit:* Provides the exact same typing experience, shortcuts, and IntelliSense as VS Code, so developers feel at home immediately.
*   **Plugin System (The "Studios"):**
    *   The IDE uses a modular architecture. **Base Studio** is just the first module. The system allows for future "cartridges" like **Solana Studio** (with Solana CLI/Anchor integration) or **Ethereum Studio**, making the IDE capable of scaling to any chain.
